FYIfor Companies · 짝꿍 기능 누락 점검

딸려 있어야 할 기능 전수 점검

"기능 A가 있으면 당연히 B도 있어야 한다"는 관점의 완결성 점검. 칸반 단계 구현·내용은 별도 검토 중이라 제외.

대부분 코드(grep/파일)로 부재 확인. 검증필요는 베이스 스키마/런타임 동작이라 코드 한 곳으로 단정 못 한 항목.

높음 운영 막힘·데이터 유실·보안 위험   중간 실사용 불편·동선 단절   낮음 있으면 좋음   검증필요
의뢰하신 예시 4가지 위치 → 로그인 뒤로가기: A-1 · 공고 승인: 이미 있음(✅ 하단) · 회원탈퇴: A-3 · 채용팀 권한 배제: 부분 있음(제외만) → 역할/오너 G-1·G-2

A인증 · 계정

#누락 기능왜 필요한가 (짝꿍 관계)심각도
A-1로그인/셋업 화면 뒤로가기·취소 동선unauthed·needs_company 화면에 로고 클릭(→/for-companies)만 있고 명시적 '뒤로/취소'가 없음. 들어왔다가 빠지는 길이 없음.중간
A-2회사 설정 단계에서 로그아웃·계정 전환잘못된 구글 계정(또는 개인메일)으로 들어오면 다른 계정으로 못 바꿈. signOut은 대시보드 진입 후 사이드바에만 존재.높음
A-3회원탈퇴 / 계정 삭제가입은 되는데 탈퇴 경로가 전무. recruiter_users·회사 연결 해제, 데이터 삭제 요청 처리 불가 (개인정보 관점에서도 필요).높음
A-4내 프로필 수정 (담당자명 등)가입 시 입력한 full_name을 이후 변경 불가.낮음
A-5OAuth 실패/취소 시 피드백구글 로그인 취소·콜백 에러 시 사용자 안내 동선 확인 필요.검증필요

B회사 · 멤버 관리

#누락 기능왜 필요한가심각도
B-1회사 설정 페이지 (회사명·로고·도메인)가입 시 1회 입력 후 고정. 회사 정보를 바꿀 화면 자체가 없음.중간
B-2회사 단위 멤버 목록·관리멤버 제외는 '공고별 면접관'만 가능. 회사에 합류한 동료 전체를 보고 내보내는 기능이 없음.중간
B-3도메인 자동합류 승인 게이트같은 회사 도메인이면 누구나 admin으로 자동 합류. 첫 가입자 외 합류를 승인/차단할 방법이 없음 — 권한·보안 리스크.높음

C공고 작성 · 관리

#누락 기능왜 필요한가심각도
C-1임시저장 (draft)'공고 게재하기' 한 번에 바로 pending_review. 작성 중 저장이 안 됨 (draft 상태는 DB엔 있으나 UI 경로 없음).중간
C-2공고 마감(closed) 버튼closed 상태는 정의돼 있는데 기업이 공고를 '마감'할 UI가 없음 (일시중지/삭제만). 채용 종료 표현 불가.중간
C-3삭제 = 하드 삭제 / 지원자 연쇄마감 대신 삭제로 유도되면 지원 이력이 사라질 위험. job_applications FK의 ON DELETE 동작 확인 필요(연쇄삭제 시 후보 데이터 유실).검증필요
C-4마감일 자동 마감deadline 입력은 받지만 지나도 자동으로 닫히지 않음.낮음
C-5채용 인원 충원 추적headcount 입력받지만 합격 인원과 연동·마감 처리 없음.낮음
C-6후보가 보는 전체 공고 미리보기작성 화면 미리보기는 미니 카드뿐. 실제 상세 페이지 형태로 못 봄.낮음

D후보 처리 (칸반 단계 외)

#누락 기능왜 필요한가심각도
D-1합격/불합격 결과 구분단계가 …→decided로 끝남. '결정'이 합격인지 불합격인지 불명확. reject 메일 템플릿은 있는데 불합격 상태/액션이 없음. (칸반 단계 재설계와 겹침 — 그쪽 검토에 합류)중간
D-2불합격/종료 후보 보관(archive)탈락 후보를 정리·숨길 방법이 없음.낮음

E메일

#누락 기능왜 필요한가심각도
E-1발송 메일 이력 화면recruiter_mail_log에 서버가 기록하고 조회 RLS 정책까지 있는데, "누구에게 뭘 보냈는지" 볼 UI가 없음. 보내고 나면 확인 불가.중간
E-2면접 일정 → 후보 자동 안내InterviewModal은 일정만 저장하고 메일과 분리. 후보는 면접이 잡힌 걸 모름. 일정 잡기와 안내 메일이 짝이어야 함.높음
E-3발송 실패/반송 상태 노출로그엔 status(sent/failed) 저장되나 기업 화면에 표시 안 됨.낮음

F면접 · 캘린더

#누락 기능왜 필요한가심각도
F-1면접 취소/삭제잡은 일정을 비우는(interview_at 해제) 경로가 없음. 수정만 되고 취소가 안 됨.중간
F-2캘린더에서 후보로 이동 / 일정 조작캘린더가 읽기 전용. 일정 클릭해도 후보 상세로 못 가고, 수정·취소 불가.중간
F-3면접 리마인더면접 전 후보/면접관 알림 없음.낮음

G채용팀 권한

#누락 기능왜 필요한가심각도
G-1역할 변경 (권한 부여/회수)초대된 팀원은 전원 interviewer 고정. 관리자 권한을 주거나 빼는 기능이 없음.중간
G-2오너 이전(transfer)공고 오너(작성자)가 퇴사·제외되면 오너 재지정이 안 됨 → 그 공고 팀 관리 불능. 오너만 팀 관리 가능하므로 치명적.높음
G-3본인이 팀 나가기면접관이 스스로 빠지는 경로 없음 (오너만 제외 가능).낮음
G-4초대 재발송메일 못 받은 경우 다시 보내기가 없음. 취소 후 재초대만 가능.낮음
G-5제외된 면접관 잔존 접근 차단job_team에서 빠진 사람이 직접 URL로 그 공고 칸반에 접근 못 하는지(RLS) 확인 필요.검증필요

H네비게이션 · 공통 UX

#누락 기능왜 필요한가심각도
H-1사이드바 설정/계정 메뉴대시보드·새공고·면접일정만. 설정·계정 진입점이 없음 (B-1·A-3·A-4의 집입구).중간
H-2에러 화면 복구 동선'접근 불가' 등에서 재시도 없이 이동 링크만.낮음

✓이미 잘 되어 있는 것 (오해 방지)
