01새 ATS 작동 흐름

[1] 회사 사람이 메인 사이트 우측 상단 "기업 채용 ↗" 작은 링크 클릭 → /for-companies 별도 랜딩
↓
[2] 랜딩에서 "공고 올리기" CTA → /company/signup 회사 이메일 입력 → Supabase 매직링크 발송
↓
[3] 메일 클릭 → /auth/callback → recruiter_companies + recruiter_users 자동 생성 → /company 대시보드
↓
[4] "+ 새 공고" → /company/jobs/new 폼 작성 (이미지 업로드 포함) → jobs 테이블에 status='live', company_id=X로 insert
↓
[5] 즉시 후보자 /jobs 피드에 노출 → 후보자 지원 → job_applications insert (status='pending')
↓
[6] /company/ats?job=X 칸반 4컬럼 (신규/열람/검토/결정) — 좌측 사이드바엔 본인 회사 LIVE 공고 🟢 dot
↓
[7] 카드 클릭 → 전체화면 모달: 중앙에 이력서 iframe + 우측 280px 패널 (정보 / 평가 메모 / 단계 이동 / 면접 일정)

06결정 포인트

① candidates 테이블에 데이터가 있는지 — 있으면 마이그레이션 후 DROP, 없으면 바로 DROP
② user_profiles.role 컬럼 — 'hr' 값을 가진 row 처리 방법 (그냥 둘지, 'seeker'로 reset)
③ isAdminUser 로직 (메인 페이지 1195행 부근) — HR 외에 admin 용도 다른 게 있는지
