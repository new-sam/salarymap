## 제품 방향 (핵심)

**회사가 self-serve로 모든 걸 처리하는 ATS**:
```
회사 가입 (이메일 매직링크)
   ↓
자기 admin에서 직접 공고 등록 (/company/jobs/new)
   ↓
후보자가 /jobs에서 보고 자동 지원 (기존 흐름)
   ↓
회사가 자기 ATS 칸반에서 들어온 이력서 직접 보고 합/불 처리
   ↓
오퍼 → 입사 → 30일 검증 → 7% 정산
```

운영(우리)이 손으로 하는 부분은 최소:
- 첫 공고 한 번 검수 (어뷰징 방지) — `/admin/jobs`에서 1클릭
- 그 외엔 회사가 자기 화면에서 처리

**"운영이 후보자 5명 큐레이션해서 보낸다"** = 콜드 영업 시 미끼 카피, 제품 기능 아님.

| # | 항목 | 선택 | 의미 |
|---|---|---|---|
| ① | 계정 분리 | **C** | 후보자 계정 ≠ 회사 계정. 한 사람이 동시 보유 불가 |
| ② | 가입 진입점 | **C** | 메인 nav "기업 채용 ↗" 작은 링크 + 별도 랜딩(/for-companies) |
| ③ | 인증 사다리 | **A** | 도메인 메일 매직링크 → draft 권한 → 발행 시 사업자번호 |
| ④ | 발행 검수 | **C** | 첫 공고만 운영 검수, 이후 자동 |
| ⑤ | Admin/ATS 구조 | **A** | Admin Shell + ATS Workspace 분리, deep link 연결 |
